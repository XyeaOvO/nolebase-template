### **1、需求分析**

本程序旨在判断一个输入字符串是否为回文字符串。所谓回文字符串，是指正着读和反着读完全一致的字符串（如“abcba”、“12321”等）。

**输入形式：**  用户从键盘输入任意字符组成的字符串，以 `#` 作为输入结束标志。

**输出形式：**  程序输出“是回文字符串”或“不是回文字符串”的判断结果。

**值的范围：**  输入字符串长度不超过 1000，字符类型为 ASCII 可见字符。

**程序功能：** 接收用户输入并判断是否为回文字符串；

**测试数据：**

- 正确输入：
    
    - 输入：`abcba#`，输出：是回文字符串。
        
    - 输入：`abcd#`，输出：不是回文字符串。
        
- 错误输入：
    
    - 输入空串或仅输入 `#`，输出：是回文字符串（可视为退化形式）；
        
    - 输入包含换行、空格等字符，程序会自动忽略换行符，不影响判断。

### **2、概要设计**

- **解决思路：**  
	利用栈后进先出和队列先进先出的性质，在输入时将元素逐个同时压入栈和队列。输入完成后，不断 pop 栈和队列，逐个比较栈顶元素和队首元素是否一致，若全部一致则为回文字符串。

- **主要数据结构：**

	- **栈 Stack：** 
		```c
		typedef struct Stack {
		    char s[MAX + 1];
		    int top;
		} stack;
		```
	    
	- **队列 Queue：** 
        ```c
		typedef struct Queue {
			char q[MAX + 1];
			int front, rear;
		} queue;
	    ```

- **主程序流程：**
	
	1. 初始化栈和队列；
	    
	2. 读取用户输入字符，直到遇到 `#`；
	    
	3. 每读取一个有效字符，压入栈、加入队列；
	    
	4. 对栈和队列进行出栈、出队比较，判断是否一致；
	    
	5. 输出判断结果。
    

- **模块调用关系**：见下函数调用关系

### **3、详细设计**

- **伪代码**（核心逻辑）： 

```
初始化空栈st，空队列q
读取字符ch，直到输入为#
    如果ch不是换行符：
        push(st, ch)
        enqueue(q, ch)

设定flag = 1
当栈非空且队列非空：
    若pop(st) != dequeue(q)
        flag = 0
        跳出循环

若flag为1，输出“是回文字符串”
否则输出“不是回文字符串”
```

- **函数调用关系图：**

```
main

├── init_Queue
├── push / pop
├── enqueue / dequeue

```

### **4、调试分析报告**

**遇到问题与解决：**

- 问题1：`scanf`在读取字符时无法正确处理换行符，导致读取中断，解决方式：改用 `getchar()` 并手动跳过换行。
    
- 问题2：循环队列边界容易出错，修正了队列满的判定条件 `(rear + 1) % MAX == front`。
    

**时空复杂度分析：**

- 时间复杂度：`O(n)`，其中 `n` 为输入字符串长度，依次压栈、入队、比较。
    
- 空间复杂度：`O(n)`，需要存储全部字符在栈和队列中。
    

**经验与体会：**
    
- 栈和队列的结合运用是解决如回文字符串这类对称性问题的经典方式；
    
- 注意输入的处理细节是程序健壮性的关键。
